Co tydzień zamawiam catering. Co tydzień ręcznie loguję się do panelu, otwieram każdy dzień i zamieniam posiłki jeden po drugim. Zajmuje to około 20 minut. W niedzielę postanowiłem, że nigdy więcej tego nie zrobię.

To opowieść o tym, jak to zautomatyzowałem — i czego dowiedziałem się o tym, jak działają współczesne aplikacje webowe.

## Problem

Panel cateringu na **panel.kuchniavikinga.pl** to React SPA (Single Page Application). Oznacza to, że HTML zwracany przez serwer jest w praktyce pusty:

```html
<body>
  <div id="app"></div>
</body>
```

Wszystko — formularz logowania, kalendarz, posiłki — renderuje JavaScript w przeglądarce. To od razu wyklucza proste narzędzia w stylu `curl` czy podstawowe skrypty HTTP. Dostaną tylko pustą „powłokę”.

## Krok 1: Uruchomienie prawdziwej przeglądarki z Playwrightem

Playwright to biblioteka Node.js, która programowo steruje prawdziwą przeglądarką Chromium — bez okna, tylko kod.

```javascript
const { chromium } = require('playwright');

const browser = await chromium.launch({ headless: true });
const page = await browser.newPage({ viewport: { width: 1440, height: 900 } });

await page.goto('https://panel.kuchniavikinga.pl/', {
  waitUntil: 'networkidle',
  timeout: 30000
});
```

Dzięki prawdziwej przeglądarce JavaScript się wykonuje, aplikacja się renderuje i mogę wchodzić w interakcję jak człowiek — tylko że w kodzie.

Logowanie było proste, gdy tylko obsłużyłem baner zgody na pliki cookie, który blokował przycisk wysłania formularza:

```javascript
// Najpierw akceptacja cookies — blokowała przycisk logowania
const cookieBtn = await page.$('text=Zezwól na wszystkie');
if (cookieBtn) {
  await cookieBtn.click();
  await page.waitForTimeout(1000);
}

await page.fill('input[name="username"]', 'user@email.com');
await page.fill('input[name="password"]', 'hasło');
await page.click('button[type="submit"]');
await page.waitForTimeout(6000); // czekanie na ustanowienie sesji
```

## Krok 2: Inżynieria wsteczna API

Tu zaczyna się ciekawie. Po zalogowaniu nie klikałem dalej przez interfejs. Zamiast tego przechwytywałem wszystkie żądania sieciowe, które robiła aplikacja:

```javascript
page.on('response', async (res) => {
  const url = res.url();
  if (url.includes('/api/')) {
    console.log(res.url());
  }
});
```

Po załadowaniu strony wypisało to około 40 wywołań API. Nazwy były w miarę oczywiste:

```
GET /api/company/customer/order/3003629
GET /api/company/general/menus/delivery/40420693/new
GET /api/company/customer/order/active-ids
GET /api/company/customer/loyalty-program/points/sum
```

Tam, gdzie nie widziałem wszystkiego automatycznie — np. co dzieje po kliknięciu „Zmień posiłek” — poszedłem głębiej. Pobrałem zminifikowany bundle JavaScript (2,5 MB) i przeszukałem go:

```bash
grep -o '/api/company/[a-z/]*' main.f4c6b44d.js | sort -u
```

Ujawniło to wzorce adresów URL, m.in.:

```
/api/company/general/menus/delivery/${t}/new
/api/company/customer/order/${e}/deliveries/${t}/delivery-meals/${n}/switch
```

Placeholdery `${t}`, `${n}` oznaczały, że ścieżki są dynamiczne — potrzebowałem tylko właściwych identyfikatorów.

## Krok 3: Kluczowy wgląd — `fetch()` wewnątrz przeglądarki

Sztuczka, która wszystko umożliwiła: zamiast wysyłać żądania HTTP z Node.js (co bez uwierzytelnienia by się nie udało), wywoływałem je **z wnętrza przeglądarki sterowanej przez Playwrighta**:

```javascript
const menuData = await page.evaluate(async () => {
  const res = await fetch('/api/company/general/menus/delivery/40420679/new');
  return await res.json();
});
```

`page.evaluate()` uruchamia kod wewnątrz Chromium. Przeglądarka ma już ciasteczka sesji po zalogowaniu — więc każde wywołanie `fetch()` jest w pełni uwierzytelnione. Serwer nie wie, że rozmawia ze skryptem.

## Krok 4: Pobranie wszystkich opcji posiłków

Skoro mogłem rozmawiać z API, pobrałem dane zamówienia, żeby znaleźć identyfikatory dostaw:

```javascript
const order = await page.evaluate(async () => {
  const res = await fetch('/api/company/customer/order/3003629');
  return await res.json();
});

// Dostawy od soboty wzwyż
const deliveries = order.deliveries
  .filter(d => d.date >= '2026-04-25' && !d.deleted)
  .sort((a, b) => a.date.localeCompare(b.date));
```

Dla każdego dnia dostawy i każdego posiłku wywołałem endpoint „switch”, żeby dostać dostępne alternatywy:

```javascript
const switchData = await page.evaluate(async ({ deliveryId, deliveryMealId }) => {
  const res = await fetch(
    `/api/company/customer/order/3003629/deliveries/${deliveryId}/delivery-meals/${deliveryMealId}/switch`
  );
  return await res.json();
```

## Krok 5: Punktacja „świadoma diety” przy wrzodziej zapaleniu jelita grubego

Mam wrzodziej zapalenie jelita grubego (WZJG) w remisji. Ogólne rady typu „zdrowe odżywianie” nie zawsze się sprawdzają — potrzebuję dużo białka, mało tłuszczu i cukru oraz unikania pewnych produktów także w remisji.

Napisałem funkcję punktacji:

```javascript
function scoreForUC(meal) {
  const n = meal.nutrition;

  // Baza: priorytet dla białka, kara za tłuszcz i cukier
  let score = (n.protein * 3) - (n.fat * 2) - (n.sugar * 1.5);

  const name = meal.menuMealName.toLowerCase();

  // Ryby przeciwzapalne — duży bonus
  if (name.includes('mintaj') || name.includes('łosoś') ||
      name.includes('dorsz') || name.includes('tuńczyk')) score += 20;

  // Chude drób — plus
  if (name.includes('kurczak') || name.includes('indyk')) score += 5;

  // Ostre potrawy mogą drażnić śluzówkę
  if (name.includes('curry') || name.includes('chili') ||
      name.includes('pikant')) score -= 15;

  // Sosy pomidorowe bywają „triggerem”
  if (name.includes('sos pomidorowy')) score -= 10;

  // Wędliny przetworzone — twardy minus
  if (name.includes('boczek') || name.includes('pepperoni')) score -= 10;

  return score;
}
```

Dla każdego posiłku wybierałem opcję z najwyższym wynikiem.

## Krok 6: Wprowadzenie zmian

Okazało się, że zmiana posiłku to pojedyncze żądanie PUT z parametrem w query:

```javascript
const result = await page.evaluate(async ({ deliveryId, deliveryMealId, dietCaloriesMealId }) => {
  const res = await fetch(
    `/api/company/customer/order/3003629/deliveries/${deliveryId}/delivery-meals/${deliveryMealId}/switch?dietCaloriesMealId=${dietCaloriesMealId}`,
    { method: 'PUT', headers: { 'Content-Type': 'application/json' } }
  );
  return { status: res.status };
}, { deliveryId: c.deliveryId, deliveryMealId: c.deliveryMealId, dietCaloriesMealId: c.bestDietCaloriesMealId });
```

Uruchomiłem to dla wszystkich 15 dni dostaw — **22 zmiany w około 30 sekund**. Zero błędów.

## Czego się nauczyłem

**1. Współczesna aplikacja webowa to dwie rzeczy: powłoka UI i API.**  
Frontend React to w dużej mierze „ładna opakowka”. Rzeczywiste dane i logika biznesowa siedzą w API. Jeśli potrafisz rozmawiać z API bezpośrednio, omijasz UI.

**2. Podsłuchiwanie ruchu sieciowego to najszybszy sposób, żeby zrozumieć aplikację.**  
Nie potrzebowałem dokumentacji. Obserwowanie tego, co przeglądarka faktycznie wysyła i odbiera, powiedziało mi wszystko w kilka minut.

**3. `page.evaluate()` to most między automatyzacją a API.**  
Uruchomienie `fetch()` wewnątrz Playwrighta oznacza, że masz uwierzytelnioną sesję przeglądarki „za darmo” — bez ręcznego zarządzania tokenami i ciasteczkami.

**4. Najtrudniejsze nie było technicznie — tylko baner cookies.**  
Szczerze: najbardziej irytujący element całego projektu.
